# 数据管理模块设计 (Data Management)

## 1. 功能概述
本模块负责数据集的导入、格式校验、增强处理及标注工具的集成。核心是将用户上传的异构数据转换为模型训练所需的标准格式（PyTorch Dataset），并提供可视化预览。
**核心增强**：必须集成一键格式转换脚本集合，支持主流格式互转及掩码生成。

## 2. ✅ 优先级
**高 (High)** - 数据是训练的基础，必须优先完成。

## 3. ⚠️ 潜在风险
- **大文件读取性能**：当导入数万张图像时，UI 可能会卡顿，需采用分页加载或懒加载机制。
- **标注格式兼容性**：用户既往数据可能包含多种格式（VOC/COCO/YOLO），解析器需具备高鲁棒性。

## 4. 🔍 确认点 (Checkpoints)
- [ ] **标注工具集成方案**：
    - 方案 A：内嵌 LabelMe（通过子进程调用，功能全但体验割裂）。
    - 方案 B：开发轻量级 Canvas（仅支持矩形框/多边形，体验一体化但开发成本高）。
    - *推荐：初期采用方案 A，后期迭代为方案 B。*
- [ ] **数据转换能力**：
    - [x] **必须支持**：JSON (LabelMe) 与 XML (VOC) 互转。
    - [x] **必须支持**：标注文件 (JSON) 生成对应的二值化 Mask (用于分割任务)。
- [ ] **数据存储策略**：
    - 选项 1：复制所有图片到项目目录（安全但占用磁盘）。
    - 选项 2：仅引用图片路径（节省空间但源文件移动会失效）。
    - *推荐：选项 1 (保证项目可移植性)。*
