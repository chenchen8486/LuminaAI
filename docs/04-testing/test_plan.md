# 测试计划 (Test Plan)

## 1. 概述
本测试计划旨在确保 LuminaAI 在各个开发阶段的质量和稳定性。

## 2. 核心链路测试点

### 2.1 数据管理模块
- **格式转换测试**：
    - [ ] **JSON 转 XML**：验证 LabelMe JSON 文件是否能正确转换为 VOC XML 格式，检查坐标和类别名称是否一致。
    - [ ] **XML 转 JSON**：验证 VOC XML 文件是否能正确转换为 LabelMe JSON 格式。
    - [ ] **无效图片拦截**：导入包含非图片文件（如 .txt, .exe）或损坏图片的文件夹，验证系统是否能正确识别并忽略/报错，不导致崩溃。
    - [ ] **二值化 Mask 生成**：验证 JSON 标注生成的 Mask 是否准确对应多边形区域，背景是否为纯黑。

### 2.2 训练引擎模块
- **UI 响应性测试**：
    - [ ] **多线程不卡死**：在训练过程中（高负载），尝试拖动窗口、点击暂停/停止按钮，验证 UI 是否流畅响应。
    - [ ] **异常处理**：模拟显存溢出（OOM）或数据损坏场景，验证 UI 是否弹出友好提示而非直接闪退。
    - [ ] **断点续训**：中断训练后，验证是否能从 Checkpoint 正确恢复并继续训练。

### 2.3 推理与导出模块
- **精度校验测试**：
    - [ ] **ONNX 导出精度**：对比 PyTorch 模型与导出后的 ONNX 模型在同一批测试图上的推理结果（logits），误差应在 1e-5 以内。
    - [ ] **过漏检统计**：使用已知 GT 的数据集进行测试，验证 FP/FN 统计结果是否与人工核对一致。
    - [ ] **批量推理性能**：测试 1000 张图片的批量推理时间，验证是否有内存泄漏。

## 3. 测试工具与环境
- **单元测试框架**：pytest
- **测试数据**：准备专门的 `tests/data` 目录，包含标准样本和异常样本。
