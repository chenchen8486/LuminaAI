# LuminaAI 深度学习视觉平台开发规范
## 1. 项目基座
- 定位：集成目标检测、分割、分类的工业级端到端视觉平台，支持本地 GPU/CPU 自适应训练与推理。
- 核心技术栈：Python (核心调度与训练), PyTorch (算法底座), C++ & OpenCV (高性能推理引擎), PySide6 (现代化前端 UI)。


## 2. 架构师绝对红线
- 拒绝硬编码：禁止针对特定测试图像或类别写特异性逻辑。
- 拒绝静默失败：模型加载失败、显存溢出或图片解析异常时，必须显式抛出异常，并在 UI 层拦截转化为友好的中文报错，禁止使用空 try-except。
- 中间态文件隔离：严格分离 data/raw_images/ (原始样本)、data/annotations/ (标注文件)、data/temp_build/ (训练中间态) 和 models/ (权重)。提供“开发者模式”开关以控制是否保留中间态文件，默认关闭。
- 工程结构整理：每次执行指令后，可能会增加脚本文件或者测试文件或一些中间结果，一定要确保工程的整洁，不能随意在任何地方生成文件或创建文件夹，一定要复合工程要求和开发审美。不需要的代码文件一定要清理，不要保留在工程中。
- 防上下文崩溃**: 在多轮代码编写过程中，请定期执行 类似上下文压缩`/compact` 命令，或自主调用以清理上下文。
- **虚拟环境**：`Chen` (已激活)
- **语言**：Python 3.10+
- **编码铁律**: 为兼容跨平台编辑器，**所有读写文本文件、JSON 文件的操作，强制使用 `encoding='utf-8-sig'`**。

## 3. 核心业务协议
- 硬件自适应：训练和推理模块初始化时，必须动态检测计算资源，自动切换 GPU 或 CPU 后备方案。
- 跨语言调用：Python 后端与 C++ 推理层的数据交互，需明确内存管理边界，避免内存泄漏。
- 训练防呆设计：面向AI小白提供友好的向导和默认配置文件，但UI必须提供“高级设置”入口，允许用户修改底层参数（如Epoch, LR, BatchSize等），并同步修改yaml配置文件。

## 4. 渐进式开发工作流 (SOP)
- 状态对齐：每次启动新任务，必须优先读取 docs/index.md 掌握全局状态，并查阅当前对应的 docs/phase_xxx.md。
- 接口先行：在开发新的标注工具或训练前，先定义好输入输出的数据结构（如 JSON 格式协议）。
- 文档同步：优化或修复问题后，必须同步更新 docs/ 目录下的开发文档，防止记忆失效。
- Git提交规范：每次完成 docs/index.md 中的一个核心任务，或者修复了一个Bug 后，需要git提交，提交信息使用中文，并清晰描述完成内容。

## 5. 交互与交付标准 (Definition of Done)
- **全程中文**: 你和我每次的对话、代码注释、GUI 提示和控制台日志输出强制使用中文。
- **文档同步**: 任务完成时，必须同步更新 `docs/` 下对应的开发文档，并在文件末尾记录变更时间与内容。
- **变更清单**: 每次执行完指令，必须在回复末尾使用 Markdown 列表，总结你本次修改或优化了哪些文件。
- **中文对话**：切记，每次和我对话一定要用中文，不要用英文。
- **pip安装**：如果需要pip请使用清华源，https://pypi.tuna.tsinghua.edu.cn/simple/
