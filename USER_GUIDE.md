# LuminaAI 详细使用说明书

本文档将指导您从零开始使用 LuminaAI 平台，详细说明数据准备、目录结构、训练流程及模型导出等各个环节。

## 1. 目录结构说明

LuminaAI 严格遵循**数据隔离原则**，所有运行时生成的数据均位于 `data/` 目录下。

### 📁 根目录结构
```
LuminaAI/
├── data/                   # [核心] 数据存储总目录
├── ui/                     # 界面源码
├── core/                   # 算法核心
├── docs/                   # 开发文档
├── runs/                   # YOLOv8 训练日志默认输出目录 (自动生成)
├── main.py                 # 启动入口
└── LuminaAI.exe            # 打包后的可执行程序 (如果有)
```

### 📁 Data 目录详解 (`data/`)
这是您最需要关注的目录，所有输入输出都在这里进行。

| 目录路径 | 用途 | 说明 |
| :--- | :--- | :--- |
| `data/01_raw/` | **原始图片存放区** | 请将您采集的原始图片（.jpg, .png）放入此文件夹或其子文件夹中。 |
| `data/02_annotations/` | **标注文件存放区** | 存放 LabelMe 的 `.json` 文件或 YOLO 格式的 `.txt` 标注文件。 |
| `data/03_temp_build/` | **训练中间态** | 系统自动生成的训练集、验证集划分文件，以及转换后的数据集结构。**通常无需手动修改**。 |
| `data/04_models/` | **模型权重存放区** | 存放训练好的模型权重文件 (`.pt`, `.onnx`)。 |
| `data/05_results/` | **推理结果输出** | 批量推理后的图片及可视化结果会自动保存在这里。 |

## 2. 快速上手流程

### 第一步：准备数据 (Step 1 & 2)

1.  **收集图片**：
    *   将拍摄好的图片放入一个文件夹（建议放在 `data/01_raw/my_dataset`，但不强制，任意位置均可）。
2.  **数据标注**：
    *   打开 LuminaAI，进入 **Step 2**。
    *   点击 **“启动 LabelMe”** 按钮。
    *   在 LabelMe 中打开您的图片文件夹，进行标注并保存 `.json` 文件（目标检测/分割）或整理文件夹结构（分类）。
    *   *注：如果是目标检测任务，LuminaAI 会在训练前自动将 JSON 转换为 YOLO 格式。*

### 第二步：导入与配置 (Step 2 & 3)

1.  **选择数据**：
    *   在 **Step 2** 点击“选择数据文件夹”，选中包含图片和标注文件的目录。
    *   系统会自动扫描并显示图片数量、标注数量及类别信息。
2.  **配置参数**：
    *   进入 **Step 3**。
    *   **基础模型**：建议初学者选择 `YOLOv8-Nano`（速度快）或 `YOLOv8-Small`（精度稍高）。
    *   **Epochs (训练轮数)**：建议设置 100-300。
    *   **Batch Size**：显存 8G 建议设为 16；显存 4G 建议设为 8；CPU 训练建议设为 4。

### 第三步：开始训练 (Step 4)

1.  点击 **“开始训练”**。
2.  **监控面板**：
    *   左侧进度条显示当前 Epoch 进度。
    *   图表区会实时绘制 Loss（损失值）下降曲线和 mAP（精度）上升曲线。
    *   右侧日志窗口会显示详细的训练信息。
3.  **训练完成**：
    *   训练结束后，最佳模型权重 (`best.pt`) 会自动保存，弹窗会提示具体路径（通常在 `runs/detect/trainX/weights/`）。

### 第四步：模型导出与推理 (Step 5)

1.  **模型导出**：
    *   在左侧面板选择刚才训练好的 `.pt` 文件。
    *   选择导出格式（如 `ONNX` 用于部署，或保持 `.pt` 用于继续推理）。
    *   点击“开始导出”。
2.  **批量推理**：
    *   在右侧面板选择**新的测试图片文件夹**。
    *   点击“运行推理”。
    *   完成后，系统会自动打开结果文件夹，您可以看到画了框（检测）或涂色（分割）的图片。
3.  **效果评估** (可选)：
    *   如果您有带标注的测试集，勾选“启用评估”并选择标注文件夹。
    *   系统将计算精确率 (Precision)、召回率 (Recall) 等专业指标，并列出漏检/误检的图片列表。

## 3. 常见问题 (FAQ)

**Q: 训练时提示“显存不足 (CUDA Out of Memory)”怎么办？**
A: 请在 Step 3 中调小 **Batch Size**（例如改为 4 或 2），或者在高级设置中减小 **Image Size**（例如改为 320）。

**Q: 我没有 NVIDIA 显卡，可以使用吗？**
A: 可以。LuminaAI 会自动检测硬件，如果没有 GPU，会自动切换到 CPU 模式。但训练速度会慢很多，建议仅用于少量数据调试或模型推理。

**Q: 标注文件一定要放在 data 目录下吗？**
A: 不强制，但强烈建议。软件会自动扫描您选择的“数据文件夹”及其子目录。为了管理方便，建议遵循 `data/` 目录规范。

**Q: 生成的 runs 文件夹越来越大怎么办？**
A: `runs/` 目录存放了每次训练的详细日志和权重备份。您可以定期清理其中不需要的 `trainX` 文件夹，只保留重要的 `weights/best.pt` 文件即可。
